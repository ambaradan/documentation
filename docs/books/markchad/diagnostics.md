## Introduzione

> Clean code is a term used to describe computer code that is easy to read, understand, and maintain.

Clean code actually encompasses more than just well-written lines of code; it represents an approach to development that prioritizes shared work; its readability allows other developers to evaluate, correct or improve the code themselves.  
In addition, when the code is easy to maintain, it is guaranteed to be improved and updated over time. This is especially important for long-term projects, such as Rocky Linux technical guides, where code must be maintained and updated for years.

The Markdown language, although not so obvious, is a programming language, and markers in the document are passed to a Markdown processor, commonly called a *parser*, which interprets them to produce a result in HTML code.
